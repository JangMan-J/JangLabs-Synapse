<objective>
Add the collision-projection primitive `project_triggers(memdir, triggers, stem=None)` to
`lib/memory_surface.py`: given a proposed `triggers` dict, synthesize matcher tokens
directly from those triggers, run them through the SAME index-walk the live `search()`
matcher uses against the on-disk catalog, and return
`{collisions, distinct_count, per_trigger}` (snake_case per D-04). Plus contract tests.

The non-negotiable correctness move (independently flagged by both the researcher and the
pattern-mapper): resolve the "no second matcher" (D-01) vs "must bypass search()'s
`_meets_min_candidate` gate" tension by EXTRACTING the shared index-walk loop out of
`search()` into one helper that BOTH `search()` and projection call. One matcher, two
callers — verifiable by grep, with zero `search()` behavior change.

Purpose: deliver the telemetry-free write-time quality signal (PROJ-01..04, QC-01) that
Phase 7 (shadow calibration) and Phase 8 (enforcement wiring) consume. This phase delivers
ONLY the primitive and its contract tests — no hook wiring, no enforcement, no read-path
change.

Output: ~3 new engine symbols in `lib/memory_surface.py` (one refactor + two additions)
and a new contract-test file `tests/memory_surface/test_collision_projection.py`.
</objective>

## Catalog/index shapes (from triggerIndex entries — test_routing_contract.py:211-225)

- Each `byCommand`/`byArg`/`bySynonym`/`byPath` value is a list of entries shaped
  `{"source": "tag"|"memory", "id": <grammar-tag-name or memory-stem>, "trigger_type": ...,
  "pattern": ...}`. `entry["id"]` is a GRAMMAR TAG NAME when `source == "tag"` (expand via
  `tagToMemoryIds`), or a MEMORY STEM when `source == "memory"` (route direct).
- `catalog["tagToMemoryIds"]` = `{grammar_tag: [stem, ...]}`.

## D-04 return shape (authoritative — snake_case, do NOT harmonize to camelCase)

The result is the three-key dict:
collisions (a list of {id: stem, via: list of {trigger: pattern, type: command|arg|path|synonym}}),
distinct_count (int — number of distinct OTHER memories matched),
per_trigger (map of raw-trigger-pattern → distinct-match-count, including 0 for non-matching triggers).

| Boundary | Description |
|----------|-------------|

## STRIDE Threat Register

This phase adds one pure-read, stdlib-only, in-process function: no network, no subprocess,
no package install, no filesystem write, no shell. ASVS L1 surface is V5 (input validation)
only. The threat surface is intentionally near-empty.

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | `triggers` dict values used as index keys | mitigate | All trigger values pass through `_norm()` (TAG_RE gate) before any dict lookup; non-conforming values are dropped, not executed. Lookups are dict-key reads, not a query language — no injection vector. (Task 2) |
| T-05-02 | Denial of Service | malformed/huge `triggers` dict raising inside the walk | mitigate | `project_triggers` is a blanket try/except returning `_EMPTY_PROJECTION` (PROJ-04 fail-open); a fault cannot crash the caller (Task 2, proven SC-4 in Task 3). |
| T-05-03 | Tampering | corpus `memory/` data mutation | accept | Projection is read-only against the compiled catalog; it opens no file for write. No mitigation needed beyond the no-write implementation; verified by the absence of any `write_atomic`/`open(...,"w")` in the new code. |
| T-05-04 | Information Disclosure | `stem` self-exclusion arg | accept | `stem` is used only as a `dict.pop` key — no path traversal, no filesystem access. Low risk; no PII. |
| T-05-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this phase — engine is stdlib-only by absolute constraint; no Package Legitimacy Gate applies. |
