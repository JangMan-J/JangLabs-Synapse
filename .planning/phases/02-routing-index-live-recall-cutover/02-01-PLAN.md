## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| memory body text → triggerIndex | Untrusted-ish store content becomes routing keys via mechanical derivation (D-29b) |
| tool_input path → refresh hook | Arbitrary file paths from tool events decide whether a rebuild fires |
| catalog file on disk | Build artifact read by every recall; a corrupt write breaks the read path |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | derive_fallback_triggers() | mitigate | Index poisoning via crafted memory body: tokens validated against `^[a-z0-9][a-z0-9._+-]{2,31}$`, generic-word exclusion, hard caps (≤6 commands/≤4 paths); derivation extracts tokens only, never executes anything |
| T-02-02 | Tampering | memory-catalog-refresh.sh path handling | mitigate | Path confusion via `..`/symlinks: keep `realpath -sm` lexical canonicalization; backing-file check uses `readlink -f` equality against the store's OWN symlink target only (mirrors proven WR-02), so an attacker-named `_grammar.md` elsewhere cannot trigger store rebuilds |
| T-02-03 | Tampering | catalog write | mitigate | Partial/corrupt catalog: existing `write_atomic()` (tmp + fsync + os.replace); `_load_catalog()` fails closed on corrupt JSON |
| T-02-04 | Denial of Service | rebuild() on grammar parse failure | mitigate | Missing/broken `_grammar.md` → `parse_grammar_md` returns `{}`, rebuild still completes (fail-open parser convention); refresh hook discards stderr and exits 0 |
| T-02-05 | Information Disclosure | routability stderr line | accept | Memory IDs (file stems) printed to stderr of a local CLI on a single-user box; no secrets in stems |
| T-02-SC | Tampering | package installs | accept | No packages installed in this phase (research: stdlib only, audit table waived) — no supply-chain surface |
