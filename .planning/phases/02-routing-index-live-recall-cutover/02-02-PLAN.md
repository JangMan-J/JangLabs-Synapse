## Phase Goal

**As a** Claude session making tool calls on this box, **I want to** have the right memories surface from a near-free precomputed evidence lookup that explains every fire, **so that** recall is trustworthy, diagnosable in seconds, and the legacy routing path can be removed with every existing memory still reachable.

This plan delivers the second vertical slice: after it completes, the user can pipe ANY tool-call payload into `python3 lib/memory_surface.py search-new` against the live store and get an explainable, evidence-routed recall response ŌĆö a capability that does not exist after Plan 02-01.

<objective>
Implement the one-matcher-over-both-levels evidence lookup (D-25) behind a staged subcommand/env dispatch (D-30), with inline evidence tuples on every surfaced memory (D-26, CORE-05), tier-based confidence gating that keeps silence the default (D-27, CORE-06), and no LLM call or new import on the read path (CORE-04). The live hook keeps calling the legacy `search` until the Plan 02-04 flip.

Purpose: CORE-04 (precomputed lookup), CORE-05 (explainable fires), CORE-06 (threshold + silence + fail-open), CORE-09 (matcher half of the contract suite).
Output: `search_new()` + helpers, extended `extract_tokens()`/`surface_text()`, `search-new` CLI subcommand, `MEMORY_SURFACE_SEARCH_IMPL` env dispatch, matcher contract tests.
</objective>
