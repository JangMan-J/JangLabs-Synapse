## Live evidence (batch a) — verified on zellij 0.45.0, this box, 2026-06-15

Probes run **serially** in the main session against two **independent zellij servers**
(separate `ZELLIJ_SOCKET_DIR`s), then torn down (operator sessions untouched).

| Probe | Result |
|-------|--------|
| Separate-server isolation | A saw 0 of B's sessions, B saw 0 of A's → independent parallel control planes, zero crosstalk |
| `run --close-on-exit` | dead-cmd pane SELF-REMOVED; same `run` without it left a lingering pane → **closes caveat #1** for tool-spawned panes |
| `run --block-until-exit` | blocked 1.5s, exit 0 → synchronous-exec primitive |
| multi-pane `subscribe -p t0 -p t1 -f json` | one stream, both pane_ids, correctly tagged, 0 leak → multiplex confirmed |
| `subscribe` delta (clean retest) | 1 initial frame + 1 delta frame carrying the marker → push-on-change |
| `paste -p` | marker 2× in target, 0× sibling → bracketed-paste, isolated |
| `send-keys -p` | `Ctrl c` landed (`^C`); text token → `exit 2 unsupported key` → control-keys only, complements write-chars |
| `clear -p` (clean retest) | t1 cleared, t0 untouched → selective |
| `watch SESSION` headless | panic rc=101 "could not enable raw mode" → needs a TTY, unusable headless; subscribe is the headless read-follow |

Two preliminary reads were REVERSED by clean retests (clear-selectivity and a
subscribe timing artifact) — recorded only the retested verdicts.
