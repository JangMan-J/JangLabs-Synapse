## Phase Goal

**As a** solo operator of this box's memory harness, **I want to** have every recall fire logged, every read-confirmation detected, and curation decided automatically from that evidence, **so that** the right memories keep surfacing with zero human curation ritual — Roulette retired and router seats machine-governed.

This plan delivers the second vertical slice: after it completes, sessions curate the store by themselves — the user starts a session, the pass runs when enough telemetry has accumulated, declineCounts move, and a one-line summary appears in the base-floor block with no human step anywhere (CUR-03).

<objective>
Build the automated maintenance pass: a `maintenance` engine subcommand that scores every memory from windowed telemetry and mutates declineCount under the D-43 rare-critical floor, a `maintenance-shadow` twin that only reports (the D-45 validation instrument), and the SessionStart trigger + summary wiring in memory-base-floor.sh (D-40/D-44) — plus the per-session telemetry marker that 03-04's evidence window counts.

Purpose: CUR-03 — periodic automated promote/demote/decay with a floor against runaway decay, no human review.
Output: engine subcommands + helpers, trigger-wired base-floor hook, MaintenancePass contract tests, and a real fixture + live demonstration of both trigger branches.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| maintenance pass → live memory frontmatter | An automated process mutates files a human authored; content loss is unacceptable (D-42) |
| SessionStart hook → session availability | A slow/hung pass would starve every session start |
| telemetry data → curation decisions | Low-quality or sparse telemetry must not cause runaway decay |
| concurrent sessions → state/telemetry files | Two sessions may pass the trigger check simultaneously |
